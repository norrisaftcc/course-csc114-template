03 / The Console Map

Five objects. One physical analogy each.

Agent
A job description. What the employee knows, what tools they can use, what resources they've been trained on.
Environment
The office. What software is installed, what the network policy is, whether outside calls are allowed.
Session
A workday. One agent, one environment, one task. When it ends, the desk is cleared. The next session starts fresh.
Vault
The key cabinet. API tokens and credentials go here. The agent never sees the raw key — the proxy handles it.
Skill
A reference manual on the shelf. A folder with a SKILL.md and supporting scripts that teach a specific capability.
04 / Ultraviolet / The System Prompt Is the Product

You're not asking a question. You're writing a constitution.

"The permanent rules this agent will follow in every conversation it ever has, with every user, forever — or until you change it and increment the version number."
§1 Identity and Role Who it is. What it's NOT.
§2 Behavioral Constraints The guardrails. Explicit refusals.
§3 Domain Knowledge Inline facts, or a mounted file path.
§4 Output Format Define the structure. Every time.
§5 Context Today's date. Who's using it. Where.
05 / The One-Change-Rule

Same outer skeleton. Three different update rules.

[initialize: v1 system prompt]
while agent not performing adequately:
    [update: change ONE thing — save as new version]
return working agent
What most people do
Rewrite the entire prompt after the first failure. Change five things at once.
∅learned
What the rule requires
Identify the ONE thing most broken. Change that. Test again. Repeat.
5×iterations
What it teaches
Causality in complex systems. Which change fixed it — and why.
∞transfers
