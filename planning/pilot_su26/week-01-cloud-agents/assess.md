# Assess — What you submit

**Slot:** Assess (graded, low stakes)
**Due:** End of Week 1, before Monday of Week 2.
**Weight:** Small — Week 1 is universal low-stakes. The point is that you *built something* and joined the GitHub class home base.

## What you submit

Both tracks (Code Builders and Prompt Masters) submit the **same artifacts** this week. Sacred Flow starts in Week 2 — this week your only GitHub job is *get a file into your repo.*

### Required artifacts, all in your `week-01/` folder:

1. **`screenshot.png`** (or `.jpg`) — one screenshot from the Claude Console showing your domain agent answering one of its "must answer" questions correctly. The screenshot should make it obvious that *your* agent (named with your initials) produced *this* response.
2. **`system-prompt-v1.md`** — the system prompt you started with (5 sections filled in).
3. **`system-prompt-vN.md`** — your latest version. If your one-change-rule iterations got you to v3 or v5, those are even better, but the latest is the one we grade.
4. **`testing-log.md`** — both the warm-up tests (Campus Info Bot) and the 5+ Domain Agent v1 tests.
5. **`tool-rationale.md`** — every tool, enabled or disabled, with a one-sentence reason.
6. **`reflection.md`** — one paragraph (3–5 sentences). Answer: *What surprised you about building this agent? What's one thing you'd change about your v1 if you had another week?*

### How to submit

| Track | How to put the files in your repo |
|---|---|
| Code Builders | Add files to your repo. GitHub Desktop, the GitHub web UI, or `git` from the terminal — your choice. **No PR required this week.** |
| Prompt Masters | Same. |

The point of letting both tracks submit by drop in Week 1 is that *the class GitHub home base is the thing that has to exist by Friday*. If you also got branches and PRs working, great. If you didn't, that's fine — Week 2 teaches Sacred Flow to Code Builders explicitly.

## What we're looking for (and what we're not)

This is **not** the place to write a portfolio piece. It's the place to **show you did the work and noticed what happened**.

### What earns full credit
- All 6 artifacts present in `week-01/`.
- Screenshot makes it obvious your agent is your own (named with your initials, in your workspace).
- The latest system prompt has all 5 sections, not just bullet points.
- `testing-log.md` shows the **3 warm-up tests + at least 5 domain tests**, each with input, expected, actual, verdict, and observation.
- `tool-rationale.md` covers every tool, even the ones you disabled.
- `reflection.md` says something specific (not "I learned a lot about agents").

### What loses points
- "Generic" system prompt with no section structure.
- Tool rationale that says "Yes" with no explanation for everything.
- Testing log that's just "it worked" / "it didn't work" with no observations.
- Missing reflection or reflection that's 1 sentence of "this was cool."

### What's *not* graded this week
- Whether your agent is impressive.
- Whether the answers were factually correct (we expect your v1 to be flawed — that's why testing exists).
- Whether you used `git` or GitHub Desktop.
- Sacred Flow process artifacts (those start Week 2 for Code Builders).

## Stretch (optional, no extra points)

If your agent is your real project for the term, your reflection can include a one-sentence note: *"I want to extend this agent with [file mounting / MCP / a specific domain expansion]. I plan to do that during Weeks 2–8 alongside the textbook work."* That signals to your instructor that you're on the agent track, which informs office-hour conversations.
